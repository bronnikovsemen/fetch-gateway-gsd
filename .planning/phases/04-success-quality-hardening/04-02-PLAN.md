    **Phase A — QUAL-01: zero `: any` types.**

    Run two complementary scans across `src/`:

    1. Strict literal scan (catches both annotation and inline-type-assertion forms):
       `grep -rnE '(^|[^a-zA-Z_]): *any( |;|,|>|$|\))' src/`
       Then strip lines that are entirely line-comments (`^\s*//`) and entirely block-comment lines (`^\s*\*`).
       Final command: `grep -rnE '(^|[^a-zA-Z_]): *any( |;|,|>|$|\))' src/ | grep -vE ':\s*//' | grep -vE '^\s*\*'`
       Acceptable count: 0.

    2. Generic-position scan (catches `as any`, `<any>`, `Array<any>` etc.):
       `grep -rnE '\bas\s+any\b|<\s*any\s*>' src/ | grep -vE ':\s*//' | grep -vE '^\s*\*'`
       Acceptable count: 0.

    If either scan returns a hit, open the offending file, replace `any` with the correct narrow type (or `unknown` + a runtime narrowing if the value truly is unstructured). Document each fix in the audit report's "Fixes Applied" section.

    **Phase B — QUAL-02: zero `console.log` statements in committed source.**

    Run two scans:

    1. Source-tree scan: `grep -rnE '\bconsole\.log\b' src/ | grep -vE ':\s*//' | grep -vE '^\s*\*'`
       Acceptable count: 0.

    2. Wider repo scan excluding `.planning/`, `node_modules/`, `.next/`, and `.git/`:
       `grep -rnE '\bconsole\.log\b' . --exclude-dir=node_modules --exclude-dir=.next --exclude-dir=.git --exclude-dir=.planning | grep -vE ':\s*//' | grep -vE '^\s*\*'`
       Acceptable count: 0.

    If either scan returns a hit in tracked source, delete the offending line (or replace with a comment if it was intentional debug scaffolding). Document each fix in the audit report.

    Note: `.planning/` is intentionally excluded — planning docs may contain `console.log` literals inside acceptance-criteria grep specifications (as in this very plan file). Only `src/`, `app/`, and other shipped-code directories are in scope.

    **Phase C — QUAL-03: no dead navigation buttons.**

    Step 1 — Enumerate every navigation call site in src/app/:
    - `grep -rnE "router\.(push|replace)\(['\"][^'\"\)]+['\"]" src/app/`

    Step 2 — Enumerate every page file under src/app/:
    - `find src/app -name 'page.tsx' -type f`
    - Compute the route path each page file corresponds to: `src/app/page.tsx` → `/`, `src/app/foo/page.tsx` → `/foo`.

    Step 3 — Cross-check:
    - Every target path from Step 1 must have a matching page file in Step 2.
    - Every page file in Step 2 must either be the root (`/`) or be reachable via at least one upstream `router.push`/`router.replace` (excluding the file's own back-reference — `/permissions` reaches `/welcome` does NOT count `/welcome`'s own back-edge as making `/welcome` reachable; the reachability check is from `/` forward).

    Step 4 — Enumerate every `<Button …>` in src/app/ that does NOT have an `onClick` handler:
    - `grep -rnE '<Button\b' src/app/`
    - For each match, read the surrounding JSX and confirm an `onClick` prop is present somewhere on the Button (multi-line JSX is allowed). Any Button without an `onClick` is a candidate dead button — investigate. (Loading-state buttons that intentionally disable themselves are NOT dead buttons; they have a real onClick that is gated by the `submitting` flag — Plan 03-01's Connect button is the canonical example.)

    Step 5 — Confirm the expected edge graph (see `<interfaces>` block) is intact. If any edge is missing or any target is missing a page file, document the gap and either:
    (a) Fix by adding the missing edge or the missing target (UNLIKELY — the upstream plans have already wired everything end-to-end), or
    (b) Halt and escalate to the user (only if the gap is ambiguous and requires a product decision).
