## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next.js router | The Connect button hands a slug into `router.push(`/connecting?provider=${selected}`)`. The slug originates from the controlled Select whose options are minted from the trusted in-app `providers` catalog — no untrusted input crosses this boundary. |
| Page unmount → pending setTimeout | A user navigating away during the 1.2s loading state must not produce a stale `router.push` on the unmounted component. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | DoS-self | Connect button setTimeout | mitigate | `useEffect` returns a cleanup that calls `clearTimeout(timerRef.current)` on unmount; the `submitting`/`!selected` guards in `handleConnect` prevent double-fire if the button is clicked again before the timer fires. |
| T-03-01-02 | Tampering | router.push target string | accept | The slug fed into the template literal is constrained at compile time to `'gusto' | 'adp' | 'paycom' | 'rippling'` via `Provider['slug']` — no untrusted input flows into the navigation target. |
| T-03-01-03 | Information disclosure | Provider catalog rendering | accept | Provider names are public marketing data; rendering them in a Select is the spec-mandated behavior. |
</threat_model>
