<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL query string → route handler | The `?provider=` value is user-controlled (could be supplied by a hostile referrer or hand-crafted URL). Must be validated against the trusted in-app catalog before being rendered or used in navigation. |
| Page unmount → pending setTimeout | A user navigating away during the 2500ms bridge must not produce a stale `router.replace` on the unmounted component. |
| Browser history stack | A transient bridge route must not sit in browser history; otherwise back-button traps occur (the same WR-01 advisory the splash deferred). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | DoS-self | Auto-advance setTimeout | mitigate | `useEffect` returns a cleanup that calls `clearTimeout(timerRef.current)` on unmount. The conditional early-return (`if (!provider) return;`) inside the effect ensures the timer only registers when the slug is valid. |
| T-03-02-02 | Tampering / Reflected XSS | `?provider=` query param | mitigate | The raw `slugParam` is NEVER interpolated directly into JSX text. Only the catalog entry's `name` (a trusted constant) is rendered. Invalid slugs short-circuit to the guard branch which renders `null` and redirects. There is no path by which untrusted query-param text reaches the DOM. |
| T-03-02-03 | History tampering / back-button trap | router navigation targets | mitigate | Both navigations on this page use `router.replace` (not `push`). The invalid-`/connecting` URL never enters history; the transient bridge state never enters history. The user's history sequence after a valid run is `/select-provider → /success` with `/connecting` absent — back from `/success` correctly returns to `/select-provider`. |
| T-03-02-04 | Information disclosure | Provider name rendering | accept | Provider names are public marketing data; rendering them in the body copy is the spec-mandated behavior. |
</threat_model>
