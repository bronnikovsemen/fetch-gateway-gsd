## Phase Goal

**As a** payroll admin landing on the Fetch Gateway demo, **I want to** navigate to every demo route and see a Fetch-branded shell with theme, font, and chrome correctly wired, **so that** the foundation is provably working before any real screen content is built on top of it.

<objective>
With the scaffold and theme already in place from Plan 01, build the three shared chrome components and the provider catalog that every screen in Phases 2-4 will consume: `FlowLayout` (centered white Paper on `#EBF5FF`), `FetchLogo` (inline SVG, size-configurable), `PermissionItem` (blue checkmark + bold label + muted description), and `src/lib/providers.ts` (four-entry catalog — Gusto / ADP / Paycom / Rippling with exact brand-color hex codes from the spec).

Purpose: These four files are the **shared vocabulary** of every later screen. If FlowLayout's spacing is wrong, every Phase 2-4 screen is wrong. If the provider catalog drifts, Phase 3's Select dropdown and Phase 3's `/connecting` provider-name copy disagree. Centralising them now — before any real screen exists — is what makes Phases 2-4 cheap.

Output: Three production-quality MUI components plus a typed provider catalog, ready to be imported by Plan 03's route stubs and by every subsequent phase.
</objective>

src/lib/providers.ts contract:
- Named export `type Provider = { slug: 'gusto' | 'adp' | 'paycom' | 'rippling'; name: string; brandColor: string };`
- Default export `const providers: readonly Provider[]` with EXACTLY these four entries in this order:
  1. { slug: 'gusto',    name: 'Gusto',    brandColor: '#F45D48' }
  2. { slug: 'adp',      name: 'ADP',      brandColor: '#D90429' }
  3. { slug: 'paycom',   name: 'Paycom',   brandColor: '#003DA5' }
  4. { slug: 'rippling', name: 'Rippling', brandColor: '#F5A623' }
- Slugs are typed as a string literal union (NOT a generic `string`). This gives Phase 3's `/connecting` query-param guard a precise discriminator.

src/components/FlowLayout.tsx contract:
- Named export `FlowLayout` (default export ALSO allowed for ergonomics, but the named one is the canonical handle).
- Props: `{ children: React.ReactNode; maxWidth?: number; padding?: number }` — `maxWidth` defaults to 440 (the welcome-screen default; Phase 2-3 plans pass other values: 768 for permissions, 498 for select-provider). `padding` defaults to 48.
- Renders an outer `<Box>` with `minHeight: '100vh'`, `bgcolor: 'background.default'` (which is `#EBF5FF` via the theme), `display: 'flex'`, `alignItems: 'center'`, `justifyContent: 'center'`, `px: 2`.
- Renders an inner `<Paper>` (MUI) with `elevation={0}` and explicit `sx={{ borderRadius: '12px', boxShadow: '0 2px 8px rgba(0,0,0,0.08)', backgroundColor: 'background.paper', width: '100%', maxWidth, p: \`${padding}px\` }}`.
- Children render inside the Paper.
- No `console.log`. No `any`. No raw `<div>` for layout.

src/components/FetchLogo.tsx contract:
- Named export `FetchLogo`.
- Props: `{ size?: number; color?: string; title?: string }` — `size` defaults to 100, `color` defaults to `'#2463EC'` (brand primary), `title` defaults to `'Fetch'` (rendered as a `<title>` element inside the SVG for a11y).
- Renders an inline `<svg>` (NOT an `<img>` tag pointing at a missing asset, NOT `@mui/icons-material/<anything>`). The SVG is a placeholder mark — a stylized lowercase "f" in brand blue, or a rounded square containing the letter F. The user will swap the actual artwork later; for now ship a recognizable placeholder. Document the placeholder nature in a JSDoc comment above the component so future maintainers know it's intended to be replaced.
- `width={size}`, `height={size}`, `role="img"`, `aria-label={title}`.

src/components/PermissionItem.tsx contract:
- Named export `PermissionItem`.
- Props: `{ label: string; description: string }`.
- Renders an MUI `<Stack direction="row" spacing={2} alignItems="flex-start">` containing:
  1. `<CheckCircleIcon fontSize="small" sx={{ color: 'primary.main', mt: '2px' }} />` (imported from `@mui/icons-material/CheckCircle`).
  2. A nested `<Box>` containing:
     - `<Typography variant="body1" sx={{ fontWeight: 700, color: 'text.primary' }}>{label}</Typography>`
     - `<Typography variant="body2" sx={{ color: 'text.secondary' }}>{description}</Typography>`
- The label MUST be bold (700 weight). The description MUST use `text.secondary` (which is `#6B7280` per the theme).
- No `console.log`. No `any`. No raw `<div>`.
